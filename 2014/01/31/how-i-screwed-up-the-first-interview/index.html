List and Tuple

  • Ref#1 Ref#2
  • Tuple is immutable; List is mutable
  • Tuple contains heterogenous sequence of elements and accessed via unpacking( a,b,c = tuple_3) or indexing; List is hemogeneous sequence and elements accessed by iterating

List Comprehensions

  • Ref#1 Ref#2
  • build a list from existing list like the way to build a set from existing set
  • example:
    • math: B: {x^2 : x in A and x is even}; python: B = [x**2 for x in A if x% 2 == 0]
    • math: C: {(x, y): x in A and y in B, x + y < 10}; python: `C = [(x, y) for x in A for y in B if x + y < 10]

Lambda

  • Ref#1 Ref#2
  • small anonymous functions
  • restricted to a single expression; can NOT contain statements

Model-view-controller (MVC)

  • Ref#1
  • Design pattern that breaks an application into three parts: data (Model), presentation layer (View), user interaction layer (Controller)
  • Event flow: User interact with application -> Controller event handler trigger -> Controller request data from Model -> Controller send data to View -> View present data to User

Implement Dictionary

data structure with (key, value) pairs

  • Asked for fasted lookup time
  • Binary search tree -> worst case take O(n) time; balanced took O(log n) time
  • Python’s dictionary is a Hash Table and is one of the mapping object
    • maps “hashable” values to arbitrary objects
