Implement layout

  • entire game board plus the label is a canvas
  • each block on the game board is 50 X 50; gameboard is 9 blocks X size_of_block + 1 (for border)
  • canvas.width = gameboard.width; canvas.height = gameboard.height + label.height
  • the radius of the circle is (3/4)/2 of the block width

Rule of Moving

  1. can move to immediate neighbouring block (8 in total) if the destination is empty
    • after the move, the piece will not be selected
  2. can jump over an immediate neighbouring block if the neighbouring block is filled and the destination is empty
    • after the move, the piece will remain selected
    • if the next click is also a jump of the same piece, then count as 1 move
  3. above are the only two valid moves

Flow

response on each click

  • click outside the gameboard: none
  • click on a block that is filled:
    • unselect previously selected piece (if there is any)
    • select the block
    • update selection
  • click on a block that is not filled:
    • check if currently selected piece can move here, if can:
      • clear currently selected block
      • draw the piece at clicked block (selected if is a jump)
      • if previous and current move are both jump, do not update moves; otherwise, add one move and update label
    • if it is an invalid move, clear the selection

Implementation - variables and algorithm

Exist

To store wether a block is filled (has a circle) or is empty, I used an object Exist which has properties 0 through 80, each corresponds to a block on the gameboar. The index of a sepecific block is calculated by col*9+row. When drawing circle on (col, row), immediately set Exist[col*9+row] as true; when clear a circle, set as false.

Selected

To store the current selected block, used object selected which has two properties col and row to determine the position. When the preperties are set to -1, it means no circle is currently selected.

CanMove

Determine whether we can move from (start_col, start_row) to (end_col, end_row)

Categories the possibilities by comparing the two column (start_col, end_col)

  • if start and end column are the same, then only case 1 and 2 (and variation) could be valid move
    1. valid move but not a jump
    2. if the position 1 is filled, then it is a valid move and jump
  • if start and end column are next to each other, then only case 3 and 4 (and variation) could be valid move
    1. valid mvoe but not a jump
    2. valid move but not a jump
  • if start and end column has one column in between (differ in 2), then only case 5 and 6 (and variation) could be valid move
    1. if position 3 is filled, then it is a valid move and jump
    2. if position 4 is filled, then it is a valid mvoe and jump
  • otherwise, it is not a valid move

Detect clicking

To detect the position where click happens, we need the event parameter, e. To get its x and y value relative to the page, we use e.pageX and e.pageY, then calculate the position relative to the canvas using e.pageX-gameboard.offsetLeft, e.pageY-gameboard.offsetTop. If the click happens on one of the block, we could then calculate the column and row number, and proceed to the rest.

We put all responses to the click into the function HalmaOnClick(e), then we need to attach this function to the canvas in order to make it happen after each click.
